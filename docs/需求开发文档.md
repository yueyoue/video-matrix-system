# 王欢项目

短视频矩阵运营系统 需求开发文档

一、项目概述

1.1 项目背景

面向短视频矩阵运营团队，解决多账号批量内容生产、自动发布、数据回收、人员工作量管理的全流程需求，替代人工逐个账号操作的低效模式，实现规模化矩阵运营。

1.2 项目目标

实现短视频批量裁切、混剪、AI 配音的自动化内容生产

支持多平台多账号定时自动发布，覆盖抖音、快手、小红书、微信视频号

自动回收各平台作品数据，支持多维度统计分析

实现多运营人员账号管理、工作量统计、生成配额管控

客户端开箱即用，支持在线更新、环境自动检测补全

提供 Web 管理后台，实现全局配置、用户、版本的统一管理

1.3 整体架构

采用 C/S 三层架构，各端职责明确：

桌面客户端：面向运营人员，提供视频处理、账号管理、发布调度、数据查看功能，需登录后使用

服务端 API：统一对接客户端请求，处理登录鉴权、配额校验、数据上报、AI 配音转发、版本检测等逻辑

Web 管理后台：面向管理员，提供用户管理、数据统计、全局配置、版本发布等管理能力

二、技术栈选型

2.1 桌面客户端

表格

技术项	选型	说明

开发语言	Python 3.11	生态成熟，视频处理、自动化库完善

界面框架	PyQt6 / PySide6	原生桌面 UI，交互体验好

视频处理	FFmpeg 静态版	负责视频裁切、混剪、编码、音频合成，单文件无依赖

自动化发布	Playwright + Chromium	模拟浏览器操作各平台后台，实现自动发布、数据抓取

打包工具	PyInstaller / Nuitka	编译为独立 exe，用户无需安装 Python 环境

安装包封装	Inno Setup	生成标准 Windows 安装包，支持桌面快捷方式

2.2 服务端与 Web 管理后台

表格

技术项	选型	说明

后端语言	PHP 8.0+	管理后台以 CRUD 为主，开发速度快，运维成本低

开发框架	ThinkPHP 6 / Laravel	内置权限、路由、数据库 ORM，快速落地

数据库	MySQL 5.7+	存储用户、账号、任务、发布记录、统计数据

文件存储	阿里云 OSS / 七牛云对象存储	存储客户端安装包、AI 配音音频文件，减轻服务器带宽压力

选型结论：PHP+MySQL 完全适配本项目场景，并发量低、业务逻辑以数据管理为主，开发效率和维护成本最优。

2.3 构建与部署

采用 GitHub Actions 实现云端自动化构建，提交代码 / 打版本标签后自动完成环境安装、依赖下载、程序打包、安装包生成

支持构建完成后自动同步安装包到国内对象存储，对接客户端在线更新

开发人员无需本地搭建完整开发环境，可通过云端完成正式版本发布

三、桌面客户端功能需求

3.1 登录认证模块

软件启动默认展示登录页，需输入用户名 + 密码验证通过后方可进入主界面

支持「记住密码」功能，本地加密存储登录凭证

登录时向服务端请求用户信息、每日生成配额、账号权限

登录失败（账号密码错误、账号已停用）给出明确提示

3.2 数据总览模块

展示当前登录用户的当日核心数据：

工作量统计卡片：今日生成视频数、发布成功数、发布失败数、管理账号数

配额提示：显示当日生成视频配额已用 / 总数

最近发布记录列表：展示平台、账号、视频名称、发布状态

异常预警：展示账号登录失效、发布失败等异常信息，支持跳转日志详情

3.3 数据分析模块

3.3.1 筛选功能

时间范围筛选：预设「今日、近 7 天、近 30 天」快捷选项，支持自定义起止日期

平台筛选：支持按抖音 / 快手 / 小红书 / 视频号筛选

账号筛选：支持按当前用户名下的具体平台账号筛选

支持导出 Excel 报表

3.3.2 数据展示

平台汇总数据：总播放量、总点赞量、总评论量、总分享量，支持对比昨日涨跌幅

分平台数据明细：按平台维度展示账号数、播放 / 点赞 / 评论 / 分享总量、平均播放量，底部展示合计行

单视频数据明细：表格展示每条视频的平台、所属账号、视频标题、播放量、点赞量、评论量、分享量、发布时间，支持分页

3.4 账号管理模块

管理当前用户名下的所有短视频平台账号，支持按平台标签切换：

支持平台：抖音、快手、小红书、微信视频号

账号列表字段：账号昵称、登录状态、作品数、总播放量、今日发布数、最后登录时间

核心操作：添加账号、批量检测登录状态、编辑账号信息、查看账号数据、删除账号

添加账号：通过内置浏览器扫码登录，自动获取 Cookie 并保存，支持 Cookie 有效期检测

3.5 视频处理模块

视频处理采用「视频库 → 裁切队列 → 篮子 → 混剪队列 → 成品」五阶段流水线架构，通过 Tab 页签切换各阶段视图，实现从原视频到成品混剪视频的全流程管理。

3.5.1 视频库（原视频管理）

支持批量上传原视频（mp4/avi/mov/mkv/flv），上传时自动通过 ffprobe 获取视频时长、文件大小等元数据

视频库展示所有视频条目，字段包含：文件名、时长、大小、来源（上传/混剪）、入库时间

支持多选勾选视频，批量添加到裁切队列

支持单个/批量删除视频

混剪完成的成品视频可一键导出回视频库，标记来源为「混剪」，便于后续发布调度

3.5.2 裁切队列

裁切组管理：将视频库中勾选的视频创建为一个裁切组，支持自定义组名

裁切规则配置：

裁切方式：支持「按段数」（将视频均分为 N 段）和「按时长」（每 N 秒切一段）两种模式

混剪取段数：配置每个混剪视频从每个来源取几段片段

裁切操作：

支持裁切选中组、裁切全部待裁切组

裁切过程展示实时进度条和当前处理文件名

裁切基于 FFmpeg 纯流复制模式（-c copy），无重编码，速度极快

裁切状态管理：待裁切 → 裁切中（百分比）→ 已完成 / 错误

支持多选删除、清空裁切队列

3.5.3 篮子（片段中间存储）

篮子是裁切与混剪之间的中间容器，每个裁切组自动关联一个篮子，裁切完成后片段自动存入对应篮子

篮子列表支持下拉切换查看不同篮子

篮子内片段展示字段：来源视频、片段序号、时长、文件路径

支持按来源视频分组查看片段，自动按片段序号排序

支持单选/多选删除片段、清空当前篮子

篮子状态：空 → 就绪（有片段）→ 混剪中 → 完成

3.5.4 混剪队列

混剪任务自动关联篮子，创建裁切组时同步创建对应的混剪任务

混剪组合逻辑：

基于 itertools.product 实现全排列组合，从每个来源视频的片段中各取指定段数进行交叉组合

支持配置「每个混剪取几段」，当取段数大于篮子中最少来源的片段数时自动降级

组合数自动计算并展示，支持预览总生成量

文字+音频组合：

支持配置多组「文字（字幕）+ 音频」组合，混剪时为每个视频分配一组

分配模式：随机分配 / 按顺序分配

音频文件支持浏览选择（mp3/wav/aac/m4a）

文字内容通过 FFmpeg drawtext 滤镜渲染为字幕叠加到视频上

混剪执行：

支持混剪选中任务、混剪全部待处理任务

混剪过程展示实时进度条（已完成数/总组合数）

混剪完成后支持一键导出到视频库

混剪状态：待处理 → 混剪中 → 已完成 / 错误

支持多选删除、清空混剪队列

3.5.5 FFmpeg 环境管理

客户端内置 FFmpeg 自动下载功能，视频处理页提供「下载 FFmpeg」按钮

自动检测顺序：本地目录 → PATH 环境变量 → 常见安装路径 → 应用目录

本地 FFmpeg 存储路径：~/.video-matrix/ffmpeg/

支持自动从 GitHub 下载 FFmpeg 静态编译包，解压 ffmpeg.exe 和 ffprobe.exe 到本地目录

下载过程展示实时进度百分比

ffprobe 兼容处理：支持 Windows 中文路径（通过临时文件方式获取输出）、shell 模式运行、CREATE_NO_WINDOW 隐藏控制台窗口

3.5.6 数据持久化

所有视频处理数据存储在本地 ~/.video-matrix/ 目录：

library/：视频库文件存储

baskets/：篮子片段文件存储（按篮子 ID 分目录）

mixed/：混剪输出文件存储（按任务 ID 分目录）

db.json：统一数据索引，管理视频库、裁切组、篮子、混剪任务的元数据

视频 ID 采用 12 位 UUID 短码，确保唯一性

3.6 发布调度模块

3.6.1 发布规则配置

可选择参与发布的平台账号

单账号每日发布上限设置

发布时间点设置：支持添加多个发布时间，按时间点自动发布

发布顺序：按视频库顺序 / 随机抽取 / 按账号分组轮询

支持「发布后自动从视频库移除」开关

3.6.2 待发布队列

展示所有计划发布的任务：计划时间、平台、账号、视频文件、状态

状态类型：等待发布、发布中、发布成功、发布失败

操作：立即发布、取消任务、批量暂停 / 恢复

3.7 接口配置模块

按平台分类展示配置项：抖音、快手、小红书、视频号

配置项包含：创作者后台地址、登录态校验接口、作品列表接口、元素选择器、Cookie 有效期等

支持恢复默认、保存配置

配置可由服务端全局下发，客户端自动同步

3.8 日志中心模块

展示系统运行全量日志：登录、视频生成、发布任务、异常报错

支持按日志级别（INFO/WARN/ERROR）筛选

支持日志搜索、一键导出

3.9 系统更新与环境检测

3.9.1 启动环境自检

软件启动时自动执行以下检测，全部通过才进入登录页：

FFmpeg 程序检测：缺失则自动从服务器下载静态版到本地目录

Playwright Chromium 内核检测：缺失则静默自动安装

系统运行库检测：检测 VC++ 运行库，缺失自动静默安装

网络连通性检测：检测服务端、AI 接口是否可正常访问

自检过程展示进度条与提示，异常给出手动修复指引

3.9.2 在线更新

侧边栏底部显示当前版本号，提供「检查更新」按钮

启动时自动检测服务端最新版本

发现新版本弹窗展示更新日志，支持「立即更新」「稍后再说」

更新过程自动下载、替换文件，完成后自动重启软件

3.10 用户信息与退出

侧边栏底部展示当前登录用户昵称、当日配额使用情况

提供「退出登录」按钮，退出后返回登录页

全局核心卡片：总用户数、今日生成视频总数、今日发布成功数、今日发布失败数、成功率

用户工作量排行：按系统用户维度统计生成数、发布成功数、失败数

平台账号统计：按平台维度统计账号数、今日发布量、成功率

4.2 用户管理模块

用户列表字段：用户名、角色、状态、每日生成配额、今日已用、最后登录时间

全局调用限制：可设置单用户每日调用上限

音色列表管理：添加 / 编辑 / 停用音色，配置音色名称、音色 ID、类型、适用场景

客户端自动同步启用中的音色列表

支持接口连通性测试

4.4 平台接口配置模块

版本发布：填写版本号、更新日志、上传客户端安装包

支持异常日志告警统计

五、核心业务逻辑说明

5.1 账号与权限逻辑

系统用户：运营人员登录客户端的账号，由管理员在后台创建，绑定每日生成视频配额，可停用 / 启用

平台账号：抖音 / 快手等短视频平台的账号，归属到某个系统用户名下，每个系统用户可管理多个平台账号

数据隔离：普通运营用户仅能查看和操作自己名下的平台账号、视频、数据；管理员可查看全局

5.2 AI 配音调用流程

客户端提交文案 + 音色 ID 到服务端接口

服务端校验用户调用配额，扣除调用次数

服务端使用统一管理的密钥调用第三方 AI 语音接口

服务端生成音频文件并存入对象存储，返回音频地址给客户端

客户端下载音频，通过 FFmpeg 合成到视频中

核心原则：AI 密钥仅保存在服务端，客户端不接触任何密钥，避免泄露。

5.3 客户端在线更新流程

客户端启动 / 手动点击检查更新时，携带当前版本号请求服务端版本接口

服务端返回最新版本信息：版本号、更新日志、安装包下载地址、是否强制更新

若版本号大于当前版本，弹窗提示用户

用户确认更新后，后台下载安装包，下载完成后自动执行覆盖安装并重启

5.4 环境自检与依赖安装流程

软件启动先进入自检流程，不展示登录页

逐项检测 FFmpeg、Chromium、运行库等依赖

依赖存在则进入下一步；不存在则自动从配置的下载地址获取，静默安装

全部检测通过后，加载登录界面

自动安装失败时，弹窗提示手动下载链接与放置路径

5.5 数据上报与同步逻辑

客户端生成视频、执行发布任务后，实时将结果上报到服务端

作品数据（播放 / 点赞等）按定时任务自动抓取并上报服务端

服务端统一存储所有数据，Web 后台与客户端均从服务端获取数据

七、非功能需求

7.1 性能要求

纯裁切视频采用流复制模式，单条视频处理耗时≤10 秒

客户端启动自检 + 登录耗时≤10 秒（首次安装下载依赖除外）

单台客户端支持同时管理 50 个以内平台账号稳定运行

服务端支持百级用户并发，接口响应时间≤200ms

7.2 兼容性要求

客户端支持 Windows 10 / Windows 11 64 位系统

管理后台兼容主流现代浏览器（Chrome、Edge、Firefox）

客户端打包后为独立程序，用户无需安装 Python、FFmpeg 等运行环境

7.3 安全性要求

用户密码加密存储，不可明文保存

AI 接口密钥、平台 Cookie 等敏感信息加密存储

客户端与服务端通信采用 HTTPS

所有接口需鉴权，禁止未授权访问

7.4 部署要求

服务端与 Web 后台可部署在单台轻量云服务器（2 核 2G 即可满足需求）

大文件（安装包、音频、素材）存放对象存储，不占用服务器带宽

支持通过 GitHub Actions 实现代码提交后自动部署到服务器
